# Contributing Guide

Thank you for your interest in MyTypist! This repository contains public product documentation and branding assets.

## How to Contribute

### Reporting Issues
- Open a GitHub issue describing the problem or suggestion.
- Include screenshots or references where applicable.
- Use the issue templates if available.

### Pull Requests
1. Fork the repository.
2. Create a feature branch (`feature/your-change`).
3. Commit your changes with a clear, descriptive message.
4. Submit a PR against the `main` branch.
5. Ensure your changes are reviewed before merging.

### What We Accept
- Documentation fixes and improvements
- Screenshot and image asset updates
- Roadmap suggestions and feature requests (via issues)
- Security vulnerability reports (see SECURITY.md)

Application code lives in private repositories. For contributions to the backend or frontend, please reach out to the maintainers directly.
